# Definition of Done (DoD)

Eine User Story gilt als **fertig**, wenn:

## Funktional
- Die Story funktioniert vollständig wie beschrieben.
- Alle definierten Akzeptanzkriterien sind erfüllt.
- Wichtige Fehlerfälle wurden berücksichtigt.

## Code & Tests
- Code baut fehlerfrei.
- Mindestens **95 % Testabdeckung** für die neue Logik.
- Alle Tests laufen erfolgreich durch.
- Code ist sauber, verständlich und ohne offensichtliche Duplikate.

## Frontend
- **Keine Farben, Schriften oder Abstände sind hardcoded** → alles kommt aus zentralen Variablen/Themes.
- UI ist konsistent mit bestehenden Komponenten.
- Fehler-, Lade- und Erfolgszustände sind sichtbar umgesetzt.

## Backend
- Eingaben werden validiert.
- Fehler werden sinnvoll geloggt.

## Dokumentation & Review
- Kurze Dokumentation oder README-Update vorhanden (falls relevant).
- Code wurde reviewed (**4-Augen-Prinzip**).
- Story ist in den Hauptbranch integriert.
